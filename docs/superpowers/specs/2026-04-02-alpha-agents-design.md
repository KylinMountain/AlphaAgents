# AlphaAgents — 新闻驱动的自主选股系统

## 概述

AlphaAgents 是一个新闻驱动的自主选股系统。监控国内外财经新闻和地缘政治事件（特别是特朗普政策动态），自主分析对A股板块的多空影响，输出推荐关注的个股列表及理由。

本系统只负责"选什么股"，不负责"怎么交易"。筛选出的个股交给已有的 TradingAgents 做单股深度分析（量价、基本面等）。

## 核心场景

1. **事件驱动**：新闻监控循环持续运行，发现重大事件时自动触发分析流程
2. **手动触发**：用户主动发起分析（如"分析今天的市场局势"、"特朗普刚发了关税声明，分析一下影响"）

## 技术栈

- **语言**：Python
- **Agent框架**：Claude Agent SDK（`claude-agent-sdk`）
- **股票数据**：akshare + baostock
- **新闻来源**：财联社、新浪财经、akshare新闻接口、海外新闻（通过WebSearch/WebFetch）

后续可替换Agent框架为开源方案，数据工具层与框架解耦。

### Agent设计

#### 主Agent（策略师）

自主决策的核心。不是固定流水线，而是根据新闻内容自主选择分析路径。

**可用工具**：
- MCP工具：`get_news`, `search_stocks`, `get_sector_data`, `get_watchlist`, `filter_stocks`
- 内置工具：`WebSearch`, `WebFetch`（实时搜索海外信息）
- `Agent`（调度子Agent）

**自主决策能力**：
- 判断新闻重要性：不重要的跳过，重大事件深挖
- 灵活分析路径：可以先查板块再搜概念，也可以直接从事件关联到个股
- 多线索追踪：一个事件可能影响多个方向（如"制裁中国"→半导体+稀土+信创），自主逐个追查
- 自选股关联：随时检查自选股是否受当前事件影响
- 信息不足时自主深挖：分析过程中如果觉得信息不够，自主决定搜索更多新闻、核查数据、检索更多概念，不是一轮就结束
- 多轮迭代分析：可以多次调用工具验证自己的判断（如先搜到"国产替代"概念，再核查该板块近期资金流向是否支持）

**系统提示词要点**：
- 新闻重要性判断框架（什么级别的事件值得分析）
- 板块影响分析方法论（美林时钟、事件→板块映射逻辑）
- 输出规范（推荐个股列表的格式和必含字段）

#### 子Agent（局势深度分析师）

仅在主Agent遇到复杂地缘政治事件时被调度。

**可用工具**：`WebSearch`, `WebFetch`, MCP工具 `get_news`

**专长**：
- 地缘政治事件深度解读（贸易战、制裁、军事冲突、外交变动）
- 特朗普政策模式识别（参考《交易的艺术》谈判策略）
- 输出：事件→板块影响映射（利好/利空 + 影响程度1-5 + 影响理由）

### MCP数据工具层

所有数据工具用 `@tool` 装饰器注册，打包为一个MCP Server。

| 工具名 | 功能 | 数据源 | 输入 | 输出 |
|--------|------|--------|------|------|
| `get_news` | 获取最新财经新闻 | 财联社/新浪/akshare | source, limit, keyword(可选) | 新闻列表(标题+摘要+时间+来源) |
| `search_stocks` | 从概念/板块/关键词检索个股 | akshare同花顺概念板块 | keyword(如"国产替代") | 匹配的概念板块名 + 成分股列表(代码+名称) |
| `get_sector_data` | 板块行情数据 | akshare | sector_name | 板块涨跌幅、资金流向、领涨股 |
| `get_watchlist` | 读取用户自选股 | 本地config/watchlist.json | 无 | 自选股列表(代码+名称+所属概念) |
| `filter_stocks` | 过滤不适合的个股 | akshare | stock_codes(列表) | 过滤后的股票列表（剔除ST、停牌、市值过小） |

**股票概念知识库（本地RAG）**：

同花顺概念板块数据量大（数百个概念、数千只股票的映射），不适合每次实时全量拉取。方案：

1. **离线构建索引**：定期（每日）从akshare拉取全量数据，构建本地索引
   - 概念板块列表（名称+描述）
   - 概念→成分股映射
   - 个股→所属概念映射（反向索引）
   - 个股基础信息（代码、名称、市值、行业）
2. **存储**：SQLite（轻量、零依赖、够用）
3. **检索**：`search_stocks` 从本地索引检索，不走网络请求

**search_stocks 检索策略**：
- 第一版用关键词匹配：对概念板块名称做模糊匹配
- 板块名称本身语义丰富（"光刻胶"、"华为概念"、"军工"），关键词匹配可覆盖大部分场景
- 不够用时再加embedding语义搜索（对概念名称+描述做向量化）

## 数据流

典型分析流程（主Agent自主决定，以下仅为示例）：

```
1. get_news() → 获取最新新闻
2. 主Agent判断：这条"特朗普宣布对华芯片制裁"是重大事件
3. 调度局势分析师 → 深度解读，输出：
   - 利好：国产替代(5)、半导体设备(4)、信创(3)
   - 利空：消费电子(3)、苹果产业链(4)
4. search_stocks("国产替代") → 概念成分股
   search_stocks("半导体设备") → 概念成分股
   search_stocks("信创") → 概念成分股
5. get_watchlist() → 自选股，检查是否有受影响的
6. 合并所有候选股
7. filter_stocks(候选股列表) → 剔除ST/停牌/小市值
8. get_sector_data("半导体") → 确认板块资金流向是否支持
9. 输出最终推荐列表
```

主Agent可能不按这个顺序，也可能跳过某些步骤（如小事件不调局势分析师），这是自主决策的体现。

## 触发方式

### CLI手动触发

```bash
# 分析当前市场局势
python main.py analyze

# 分析特定事件
python main.py analyze --event "特朗普宣布对华加征关税"

# 检查自选股是否受近期事件影响
python main.py watchlist-check
```

### 事件驱动（新闻监控循环）

```bash
# 启动监控，持续运行
python main.py monitor --interval 300
```

每隔N秒拉取新闻，与已处理新闻去重，发现新的重大事件时自动触发分析。

## 第一个里程碑（MVP）

能自主运行完整闭环：
1. 新闻监控持续运行，实时拉取新闻
2. 自主判断新闻重要性，过滤噪音
3. 重大事件自动触发分析流程
4. 分析板块影响
5. 检索相关个股
6. 过滤筛选
7. 自动输出推荐列表

MVP阶段简化项：
- 新闻源先只接akshare（接口稳定），财联社爬虫后续加
- `search_stocks`先用关键词匹配，不做embedding
- 输出只做终端文本
- 手动触发作为补充（`python main.py analyze --event "..."）`
